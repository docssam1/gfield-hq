## 2. 지금 프로그램의 기본 구조

지금 프로그램은 다음 흐름으로 움직입니다.

1. 인트로 영상
2. 로그인/승인 화면
3. 로그인 후 인사 연출
4. 바로 아래에 `ebook 뷰어`
5. 그 아래에 `강의 버튼 / 자료 다운로드`
6. 그 아래에 `DOCSSAM 전용 AI 튜터 소개`
7. 그 아래에 `브랜드/플랫폼 소개`

즉 지금 기준의 첫 화면은 소개 페이지가 아니라, `강의와 AI가 붙어 있는 ebook 뷰어`입니다.

## 3. 관리자 화면에서 할 수 있는 일

관리자 화면에서는 다음 항목을 수정할 수 있습니다.

### 기본 설정

- 과정명
- ebook 뷰어 링크
- AI 튜터 사용 여부
- AI 튜터 링크

### 승인 학생 목록

- 승인 학생 이름 불러오기
- 수정하기
- 저장하기

한 줄에 한 명씩 넣으면 됩니다.

예시:

```text
노관호
김명서
이하린
```

### 상위 버튼 관리

상위 버튼은 책 안의 큰 분류입니다.

예시:

- 개념
- 필수유형
- 실전 모의

각 상위 버튼마다 넣을 수 있는 값:

- 상위 버튼 이름
- 보조 캡션
- 이미지 주소
- 버튼 스타일 5종 라디오 선택

현재 스타일 5종:

- classic
- gold
- navy
- emerald
- wine

### 강의 버튼 관리

각 강의 버튼은 다음 구조로 저장됩니다.

- 상위 버튼
- 버튼 이름
- 페이지
- 링크
- 스크롤 타깃
- 다운로드 라벨
- 다운로드 링크

예시:

```text
상위 버튼: 개념
버튼 이름: 필수유형 1
페이지: 1.5
링크: https://...
스크롤 타깃: #p1-5
다운로드 라벨: 개념 노트
다운로드 링크: ./files/note-1.pdf
```

## 4. 페이지 값은 어떻게 쓰는가

`페이지`는 종이 A4 페이지 번호만 뜻하지 않습니다.

이 값은 실제 화면에서 보이는 구간을 나누는 기준으로도 씁니다.

그래서 다음처럼 써도 됩니다.

- `1`
- `1.5`
- `2`
- `2.5`

이 값은 학생에게 직접 설명하는 값이 아니라, 버튼 묶음과 스크롤 흐름을 정리하는 내부 기준입니다.

## 5. 강의 버튼은 어떻게 동작하는가

강의 버튼을 누르면 현재는 다음 동작이 가능합니다.

1. 연결된 강의 링크 새창 열기
2. 연결된 스크롤 타깃 위치로 이동
3. 다운로드 링크가 있으면 자료 다운로드

즉 버튼 하나에

- 강의
- 스크롤
- 다운로드

를 함께 걸 수 있습니다.

## 6. ebook 링크는 어떻게 연결하는가

관리자 화면의 `ebook 뷰어 링크` 칸에 경로를 넣으면 됩니다.

예시:

```text
./books/hs-thinking-basic/index.html
```

또는

```text
./files/hs-thinking-basic.pdf
```

연결되면 로그인 후 첫 화면의 ebook 영역에 바로 뜹니다.

연결 전에는 안내 프리뷰 카드가 보입니다.

## 6-1. ebook 동작 명칭 정리

앞으로는 ebook 연결 상태를 이렇게 부릅니다.

### 1. ebook 단독 모드

- 교재 파일만 연결된 상태
- 영상 링크는 없음
- 버튼과 AI 튜터는 있을 수 있음
- 자료 다운로드 버튼도 함께 둘 수 있음

### 2. 강의 연동 모드

- 교재 파일 + 영상 링크까지 함께 연결된 상태
- 버튼을 누르면 강의 보기, 스크롤 이동, 자료 다운로드가 같이 작동할 수 있음

## 6-2. 앞으로 들어갈 ebook 뷰어 보기 모드

ebook 뷰어 자체는 차후 다음 두 보기 모드로 정리합니다.

### 1. 한 페이지 보기

- 한 페이지씩 보는 모드
- 모바일에서는 세로 스크롤만이 아니라 손으로 밀어 넘기는 동작도 고려

### 2. 책으로 보기

- 양면 2페이지가 펼쳐진 책처럼 보는 모드
- 오른쪽 탭 시 다음 페이지
- 왼쪽 탭 시 이전 페이지
- 흐린 화살표로 방향 표시
- 페이지 넘김 효과 추가

중요:

이 기능은 지금 바깥 아카이브 화면이 아니라, `ebook 뷰어 페이지 내부`에서 구현해야 합니다.  
즉 현재 메인 아카이브는 뷰어를 연결하는 역할이고, 실제 손밀기/탭 넘김/책장 효과는 연결된 ebook HTML 또는 전용 viewer 안에서 작업하는 단계입니다.

## 7. AI 튜터는 어떻게 연결되는가

AI 튜터 버튼은 학생이 다시 이름을 입력하는 구조가 아닙니다.

로그인 시 이미 저장된

- 학생 이름
- 과정명

을 기준으로 이어집니다.

즉 현재 설계 기준은:

- 버튼에서 이름과 과정 정보를 이미 갖고 있음
- GPT는 그 다음에 몇 쪽인지 묻는 구조

입니다.

## 8. 지금 저장되는 위치

지금은 서버 DB가 아니라, 브라우저의 `localStorage`에 저장됩니다.

키 이름:

```text
gfield_archive_admin_config
```

즉 같은 PC, 같은 브라우저에서 열면 저장값이 유지됩니다.
