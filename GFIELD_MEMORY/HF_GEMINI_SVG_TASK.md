## ⛔ 최우선 원칙 — 유사문항의 정의 (이것을 어기면 전부 무효)

```
유사문항 생성 ≠ 새 문제 만들기
유사문항 생성 = 원본 구조 100% 복제 + 숫자만 변경
```

- 원본 이미지의 도형 배치, 칸 구조, 보는 방향, 문제 형식을 그대로 복제한다.
- 오직 숫자(개수, 조건값)만 바꾼다.
- "비슷한 다른 문제"를 만들지 마라. "같은 문제의 숫자만 바꾼 버전"을 만들어라.
- 풀이 방식과 문제 유형은 원본과 100% 동일해야 한다.
- 원본에 쌓기나무 6개면 → 유사문항도 같은 모양 배치에 개수만 다르게.
- 새로운 발상, 새로운 조건, 새로운 도형 추가 절대 금지.

이 원칙을 어긴 산출물은 전부 폐기 대상이다.

## 배치 처리 (반드시 이 단위로)

54개를 5개씩 11배치로 나눈다.

```
배치 1: typeId 01~05
배치 2: typeId 06~10
배치 3: typeId 11~15
배치 4: typeId 16~20
배치 5: typeId 21~25
배치 6: typeId 26~30
배치 7: typeId 31~35
배치 8: typeId 36~40
배치 9: typeId 41~45
배치 10: typeId 46~50
배치 11: typeId 51~54
```

각 배치마다:
```
1. 해당 5개 이미지를 직접 본다.
2. 각 이미지의 숫자/조건/도형 구조를 읽는다.
3. hf_data.json에서 해당 typeId의 정답/스크립트를 참고한다.
4. 유사문항 SVG 2개를 만든다 (숫자/조건만 변경).
5. 만든 유사문항의 정답을 직접 풀어서 검산한다.
6. 검산 통과한 것만 출력한다.
7. 다음 배치로 넘어가기 전, 이번 배치 5개가 완료됐는지 확인한다.
```

한 배치 끝나면 "배치 N 완료 (typeId XX~XX)" 라고 보고하고 다음 배치 진행.

## SVG 생성 규칙

```
- 원본 이미지의 핵심 도형/구조를 그대로 유지
- 숫자/조건만 바꿔라 (그림 형태는 유지)
- viewBox: "0 0 400 300"
- stroke 기반 path 위주 (Vivus.js 손글씨 재생 가능하도록)
- 색상: 기본 #0d1b2a, 강조 #e9c176 (브라스골드)
- 그림 문제는 반드시 그림으로 (텍스트로 바꾸기 절대 금지)
- 텍스트가 필요하면 <text> 요소 사용
- 금지어: 교차점, 단면화, 단면, 교차 (유아어로 대체)
```

유아어 대체:
```
교차점 → 만나는 곳
단면 → 잘린 면
```
