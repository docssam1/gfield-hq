# GFIELD-ON Archiv 파일럿 — 홈페이지 / 로그인 하단 / 플랫폼 홈 인수인계

> 프로젝트명은 **GFIELD-ON Archiv 파일럿**으로 고정한다.  
> 이 문서는 Claude 및 후속 작업자에게 넘기는 홈페이지·플랫폼 홈 기준이다.  
> 정식 서비스를 전부 만드는 것이 아니라, 정식 서비스처럼 보이는 파일럿 흐름을 만든다.

---

## 1. 역할 구분

### GPT 담당

```text
gfield-on-archive-preview.html
= 소개 화면 + 로그인 + 로그인 이후 플랫폼 홈
```

GPT가 잡는 영역:

- 로그인 창 아래 홈페이지 내용
- 지필드 10년 역사 / 철학 / 시크릿 서재 소개
- 로그인 이후 플랫폼 홈 구성
- Smart eBook 진입 버튼
- 교육정보 / 유튜브 / 인사이트 카드 영역
- 자료실 / 리포트 / 공지 등 플랫폼 입구

### Claude 담당

```text
gfield-on-archive-ebook-library.html
= Smart eBook 서재

books/hs-thinking-basic/viewer-v2.html
= 영상 + 교재 뷰어 URL 파라미터 확인
```

Claude가 잡는 영역:

- Smart eBook 서재
- 권한 있는 책 목록 표시
- 책 선택 후 3D 책 설명 영역
- [이북 강의 보기] → viewer-v2 이동

## 2. 전체 화면 흐름

```text
1. 소개 화면
   - 지필드 10년의 역사
   - 성공하는 공부 습관의 비밀
   - 승인된 학생만의 시크릿 서재

2. 로그인 영역
   - 학생 이름
   - 승인번호
   - 입장하기

3. 로그인 이후 플랫폼 홈
   - Smart eBook
   - 자료실
   - 교육정보
   - 학습 리포트
   - 공지
   - 작은 AI-DOT 말풍선

4. Smart eBook 서재
   - 책만 보이는 서재
   - 파일럿에서는 모든 책이 보이는 것으로 처리

5. viewer-v2
   - 영상 + 교재 동시 보기
```

---

## 3. 소개 화면은 유지한다

소개 화면은 지우면 안 된다.  
기능 링크를 덕지덕지 다는 영역도 아니다.

핵심 카피:

```text
어느 누구보다 앞서 준비해 온
지필드의 비밀 서재가 열립니다.
```

```text
성공하는 공부 습관의 비밀,
지필드의 10년이 GFIELD-ON Archiv로 이어집니다.
```

포함할 내용:

- 14년 전 전자펜 첨삭
- 맞춤 영상 설명
- 한 문제 한 문제 확인하던 기록
- 지필드는 기술이 아니라 “어떻게 가르칠까”를 고민해 왔다는 철학
- 승인된 학생만을 위한 시크릿 공간

---

## 4. 로그인 이후 플랫폼 홈에 들어갈 영역

로그인 이후 플랫폼 홈은 기능 입구다.  
여기에는 Smart eBook뿐 아니라 상담 화면에서 보여줄 교육 정보도 들어간다.

### 4.1 교육 정보 카드 영역

제목:

```text
상담 화면에 보여줄 교육 정보
```

설명:

```text
뉴스 큐레이터·유튜브·인사이트는 홈에서 보여주고,
실제 학습은 뷰어로 이어집니다.
```

카드 1:

```text
YOUTUBE
문제푸는 방법 찾기 학습 흐름
교재와 연결될 수 있는 대표 교육 영상 링크입니다.
```

카드 2:

```text
INSIGHT
DOCSSAM INSIGHT
교재는 파일이 아니라, 영상·질문·자료로 이어지는 학습 흐름입니다.
```

카드 3:

```text
NEWS
대치동 교육 정보
상담 시 학부모에게 보여줄 수 있는 교육 브리핑 카드 영역입니다.
```

이 영역은 Smart eBook 서재 안에 넣지 않는다.  
반드시 로그인 이후 플랫폼 홈에 둔다.

### 4.2 플랫폼 입구 카드

플랫폼 홈에는 다음 입구가 있어야 한다.

```text
Smart eBook
- 영상이 연결된 책 서재로 이동

자료실
- 일반 자료 / 공지 / 학습 파일

교육정보
- 뉴스 큐레이터 / 유튜브 / 인사이트

학습 리포트
- 향후 연결 예정
```

AI-DOT은 작은 말풍선만 허용한다.

```text
막히면 쪽수와 문제 번호부터 확인할게요.
```

## 5. Smart eBook 영역 구분

Smart eBook은 책 전용 서재다.

들어갈 것:

- 책 표지
- 교재명
- 과목
- 과정
- 영상 연결 여부
- 자료/정답 여부
- 책 선택
- 3D 책 설명
- [이북 강의 보기]

넣지 말 것:

- 뉴스
- 교육정보
- AI-DOC 큰 소개
- GitHub 게시판
- 미러링 관제 문구
- 상담 CTA
- 학부모 반응
- 일반 자료실

흐름:

```text
플랫폼 홈
→ Smart eBook 클릭
→ gfield-on-archive-ebook-library.html
→ 책 선택
→ 3D 책 설명
→ 이북 강의 보기
→ viewer-v2
```

## 6. 홈페이지 하단 구성 제안

로그인 창 아래에 다음 순서로 둔다.

```text
A. GFIELD-ON Archiv 철학 섹션
- 10년의 기록
- 성공하는 공부 습관의 비밀
- 전자펜·첨삭·맞춤 영상의 역사

B. 로그인 이후 플랫폼 프리뷰 섹션
- 자료실 이미지 / 플랫폼 목업 중심
- 수학·영어·과학 확장감
- Smart eBook / 자료실 / 교육정보 / 리포트 진입 카드

C. 상담 화면에 보여줄 교육 정보 섹션
- YOUTUBE / INSIGHT / NEWS 3개 카드

D. Smart eBook 소개 섹션
- 책만 관리되는 시크릿 서재
- [Smart eBook 입장] 버튼

E. 작은 AI-DOT 말풍선
- 큰 기능 설명 금지
```

---

## 7. 금지

- 채팅방/프로젝트 이름 임의 변경 금지
- “지필드-ON”으로 바꿔 부르지 않기
- 소개 화면 삭제 금지
- 로그인 이후 플랫폼 홈을 Smart eBook 서재와 섞지 않기
- Smart eBook 서재에 뉴스/AI-DOC/미러링 관제 넣지 않기
- GitHub 게시판 사용자 노출 금지
- 기존 viewer.html 수정 금지
- gfield report 3 운영본 접근 금지
